1. 日常启动
直接打开 VS Code 载入工程文件夹即可。

点击底部状态栏的 Build 直接编译。

2. 绝对禁忌
不要手贱：千万别点 VS Code 底部状态栏的“编译器/Kit”选项。一旦点了，它就会覆盖你的完美配置，跑去抓取最新的 14.42 报错。一切以 .vscode/settings.json 为准。

3. 终极急救法（遇到任何 CMake 玄学报错时）
只做两步，绝不废话：

物理删除 工程目录下的 build 文件夹（清除旧缓存）。

在 VS Code 按 Ctrl + Shift + P，运行 CMake: Configure。
